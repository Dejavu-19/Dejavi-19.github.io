面试的流程

面试官谈面试

应届生—–最看重的还是编程语言、数据结构与算法的基本功。要求新人拥有厚重的项目经验是不太现实的，project只能作为锦上添花的事。最关键的还是手撕代码的能力，以及与面试官交流时体现的沟通能力、学习能力和逻辑思维能力。

不要基于编写代码，最好跟面试官多沟通，然后开始做一些整体的规划和设计，有助于编写高质量和高可读性的代码。写完后不要马上提交，最好自己检查并借助一些测试用例来测试几遍代码，找出可能出现的错误。

应聘者要事先做好准备，对公司近期动向，项目情况有所了解，对所应聘的岗位怀有热情。另外可以提前准备好一些问题问面试官。

面试的三种形式

1. 电话面试：

没有肢体语言、面部表情，清楚表达自己的想法会比现场面试更难。要用形象化的语言尽可能把技术细节说清楚。eg. 现场面试二叉树可以画图，电话的话需要把哪些节点，每个节点左右节点分别是什么说清楚。

对应聘者英语听力能力提出了更高要求。如果没有听清楚或听懂面试官问题，要敢于说Pardon，不要不懂装懂、答非所问，这是大忌。不确定时一定要大胆向面试官多提问，知道弄清楚面试官意图为止。

2. 共享桌面远程面试(Phone-Screen Interview)

这种形式的面试，面试官最关心面试者编程习惯和调试能力。

  • 思考清楚再开始编码。策略是想清楚解决问题的思路，算法的时间、空间复杂度各是什么，哪些特殊情况需要处理等，然后再开始编写代码。

  • 良好的代码命名和缩进对齐习惯。

  • 能够进行单元测试。单元测试=证明自己有专业的软件开发经验。如果能做到测试在前、开发在后更牛逼。

    调试能力是面试官关注的重点。如果能熟练的设置断点、单步跟踪、查看内存、分析调用栈，就能很快地发现问题根源并解决，那么面试官会觉得应聘者开发经验很丰富。调适能力是书本上学不到的，只有通过大量的软件开发时间才能积累出调试技巧。

3. 现场面试Onsite Interview

准备工作：

  • 规划好路线并估算出行时间。半小时缓冲时间防止迟到。

  • 准备好得体的衣服。

  • 注意面试邀请函中的面试流程。

  • 准备好几个问题。

    现场面试是面试流程中的重头戏，通过应聘者的语言和行动考察沟通能力、学习能力、编程能力等综合能力。

面试的三个环节

通常每一轮面试分为三个环节：

  • 行为面试：参照resume了解应聘者过往经验。
  • 技术面试：要求应聘者现场写代码。
  • 应聘者提问：问几个自己感兴趣的问题。

1. 行为面试环节

面试开始5~10min内通常为行为面试。面试官会注意应聘者的性格特点，深入了解简历项目。

不少面试官会让应聘者做一个简短的自我介绍。由于对方已经有你的resume，用30s~1min的时间介绍自己的学习、工作经历即可。如果面试官对你某一段经历或项目感兴趣会针对性的提几个问题详细了解。

  • 项目经验

    自我介绍后，面试官会对应resume去了解他感兴趣的项目。采用STAR模型。

    Situation————Task————Action————Result

    简短项目背景————自己完成的任务————为了任务做了哪些部分用了什么技术————自己的贡献与量化数字体现

    • Situation简短项目背景。比如项目规模，开发的软件功能、目标用户等。

    • Task自己完成的任务。这个要写详细，让面试官对自己的工作一目了然。用词上注意区分参与和负责，

      面试官如果看到负责可能会问你项目总体框架设计、核心算法、团队合作等问题。这些问题对于仅是简单参与的人来说是很难回答的，会让面试官认为你不诚实，印象分减去很多。

    • Action做了哪些部分工作，是怎么做的。这里可以详细介绍。eg 系统设计——可以介绍系统架构的特点；软件开发——可以写基于什么工具在哪个平台下应用了哪些技术；软件测试——手工测试还是自动化测试、白盒测试还是黑箱测试等等。

    • Result自己的贡献。信息可以写的具体一些，最好用数字加以说明。eg 功能开发，则可以说按时完成了多少功能；优化，性能提高百分比是多少；维护，可以说修改了多少个bug。

    如果上述模型信息还不够清晰，面试官则可能问几个相关的问题：

    • 你在项目中碰到的最大的问题是什么？是怎么解决的？

    • 从这项目中你学到了什么？

    • 什么时候会和其他成员有什么样的冲突，你们是怎么解决冲突的？

      对每一个项目精力都做好相应的充足准备，才能做到游刃有余。介绍项目经验时(包括resume和面试时口头介绍)不要详述项目背景，都要突出自己完成的工作与取得的成绩。

  • 应聘者掌握的技能

    描述即能掌握程度时，注意了解、熟悉和精通的区别。

    • 了解——上过课看过书，但是没有做过实际项目，不建议列在简历里。

    • 熟悉——简历中描述的技能的掌握程度应该是熟悉，对应届生而言，毕业设计中用到的技能可以用熟悉来描述；实际项目长时间的某项技术，可以通过查阅文档独立解决大部分问题；已经工作的，在项目开发中所用到的都可以成为熟悉。

    • 精通——可以轻松的回答这个领域中的绝大多数问题

2.技术面试环节

通过简历及行为面试大体了解背景后，接下里就是技术面试。一轮1h的面试技术面试通常会占40-50min，这是面试的重头戏，对面试结果起决定性作用。

关注应聘者五个素质：

  • 基础知识扎实全面，包括编程语言、数据结构、算法等
  • 能写正确的bug-free、完整的、鲁棒的代码
  • 能思路清晰得分析、解决复杂问题
  • 能从时间和空间两方面优化算法效率
  • 学习、沟通、发散思维能力等

简单问题——注意细节，写出完整的、鲁棒的代码

复杂问题——可以通过画图、举具体例子和分析复杂度理清楚思路后再动手编程，力求最优解

主动提问——弄清楚题目要求，展现自己的沟通能力

当面试官前后问的两个有相关性时候，尽量把解决前面问题的思路迁移到后面问题中，展示自己良好的学习能力。

  1. 扎实的基础知识：编程语言、数据结构和算法。

    首先，熟练掌握1到2门编程语言。

    其次，面试前熟练掌握链表、树、栈、队列和哈希表等数据结构 eg.重点注意链表插入删除节点，二叉树的各种遍历方法的循环和递归写法。

    最后，了解各种查找排序等算法基础上，eg. 重点注意二分查找、归并排序和快速排序，少数注重算法的公司如百度、谷歌还会要求应聘者熟练掌握动态规划和贪婪算法

  2. 高质量的代码

    边界调节、特殊输入等细枝末节要额外关注。要把问题方方面面都考虑到做好defence。

    最好的解决办法是动手写代码前想好测试用例。只有把所有可能的事先输入都想好了才能针对各种情况做出相应处理。

    面试官出了应聘者能写出具有基本功能的代码外，还会关注应聘者是否考虑了边界条件、特殊输入(如nullptr指针、空字符串等)及错误处理。

  3. 清晰的思路

    几个方法帮助形成思路：

    举例举一两个例子模拟操作的过程；

    画图尝试用图形表示抽象的数据结构如链表、二叉树；

    分解最后可以试着把复杂的问题分解成若干个简单的子问题，如基于递归思路的分治法、动态规划等。

  4. 优化效率的能力

    想优化代码的效率，需要熟知各种数据结构的优缺点，并选择合适的数据结构解决问题；以及常用的算法(排序和查找)。

  5. 优秀的综合能力

    Soft Skills语言表达能力，介绍项目经验或者算法思路时观点明确、逻辑清晰=沟通能力强。

    最近看什么书，学到什么新知识=学习能力强。抛出一个新概念(丑数)，看应聘者能不能经过提问、思考、再提问总结出规律并找到解决方案。

    知识迁移能力、抽象建模能力、发散思维能力等

应聘者提问环节

每一轮面试提前准备1~2个问题

不要问和自己职位没关系的问题，不要问薪水，不要问面试结果。

推荐问职位或项目相关的问题：

  • 提前搜集信息，对公司成立时间、主要业务、职位要求都了然于胸
  • 面试过程中留心面试官说过的话，其中可能包含其他渠道无法得知的信息，比如项目进展等，

03. 剑指 Offer 03. 数组中重复的数字

找出数组中重复的数字。

在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。

示例 1：

输入：
[2, 3, 1, 0, 2, 5, 3]
输出：2 或 3

限制：

2 <= n <= 100000

这道题在原书上绝对不是简单级别。
它考察的是程序员的沟通能力，先问面试官要时间/空间需求！
只是时间优先就用字典，
还有空间要求，就用指针+原地排序数组，
如果面试官要求空间O(1)并且不能修改原数组，还得写成二分法！
