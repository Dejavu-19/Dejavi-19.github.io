✅167. Two Sum II - Input array is sorted (Easy)

Given an array of integers that is already sorted in ascending order, find two numbers such that they add up to a specific target number.

The function twoSum should return indices of the two numbers such that they add up to the target, where index1 must be less than index2. Please note that your returned answers (both index1 and index2) are not zero-based. You may assume that each input would have exactly one solution and you may not use the same element twice. Input: numbers={2, 7, 11, 15}, target=9 Output: index1=1, index2=2

Solution1 :

要求：you may not use the same element twice.

  1. 思路：夹逼法。两边开始，如果两数和大于目标，则右侧指针左移，反之左侧指针右移。
  2. two pointer O(n logn) O(1) good sorting algorithms like Quicksort or Merge sort or Heapsort O(n logn)
  3. Let the two pointers i and j correspond to the head and tail of the array, respectively, and sum the two. num[i]+num[j] = k
  4. If k is greater than traget, perform j-1 operation (because of ascending order, the tail number is large);
    If k is smaller than target, perform i+1 operation.
  5. Return the index

  1. 思路：双指针法。思路与上一题一致，只不过需要跳过非元音字母。注意大小写。

  2. Using two pointers, one pointer traverses from head to tail, and one pointer traverses from end to head. When both pointers traverse to vowel characters, the two vowel characters are swapped.

Note: Do not modify the linked list. Follow up: Can you solve it without using extra space?

  1. 思路：双指针技术。
  2. 假设慢指针到达 Loop 起点时，走了 k 步，则快指针已经在 Loop 中 k 步。
  3. 两点首次相遇， 离开 Loop 起点 loop size – k 步，距离 loop 起点 k=(L- (L-k))步，所以再用一指针从头开始，与慢指针一起一步一 步移动，再相遇时就是起点。
  4. Time complexity : O(n)O(n)
  5. Space complexity : O(1)O(1)

42. Trapping Rain Water🎂✨

Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it is able to trap after raining. For example, Given [0,1,0,2,1,0,1,3,2,1,2,1], return 6

The above elevation map is represented by array [0,1,0,2,1,0,1,3,2,1,2,1]. In this case, 6 units of rain water (blue section) are being trapped. Thanks Marcos for contributing this image!

Solution1:

  1. 思路：夹逼法。Using 2 pointers
  2. 两边先去除比中间小的。然后两边哪边小哪边就往中间移动，因为另一侧高，水位不会高于较小的一侧。移动中碰到更大的则停止当前轮，碰到更小的则添加相应的值到结果中，左边的就是左边起始点的 值-当前值，右边的是右边起始点的值-当前值。
  3. Time complexity: O(n).
  4. Space complexity: O(1) extra space. Only constant space required for \text{left}left, \text{right}right, \text{left_max}left_max and \text{right_max}right_max.

925. Long Pressed Name

Solution1:

  1. Two Pointer
  2. This leads to the following algorithm:
    • First, we run a loop to move the two pointers along the strings, until we reach the end of either string.
      • For each character in name, if there’s a match with the next character in typed, we advance both pointers.
      • If they are a mismatch, and it’s the first character of the block in typed, the answer is False.
      • Else, discard all similar characters of typed coming up. The next (different) character coming must match.
    • At the end of the loop, we would end up three cases:
      • case 1). if there is still some characters left unmatched in the name string, then we don’t have a match.
      • case 2). if there is still some characters left in the typed string, and all the remaining characters are resulted from the long press, then we still have a match.
      • case 3). otherwise, if any of the remaining characters in the typed string is not redundant, then we don’t have a match.
  3. Time Complexity: O(N+T)O(N+T), where N, TN,T are the lengths of name and typed.
  4. Space Complexity:
    • For the Python solution, we have O(1)O(1) space complexity.
    • For the Java solution, we need additional space due to the conversion made by .toCharArray() function. Hence, the space complexity would be O(N+T)O(N+T).
