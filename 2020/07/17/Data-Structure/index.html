Time complexity and space complexity

  1. The faster an algorithm runs,the better its time complexity is and therefore the better the algorithm is.
  2. Similarly, the less memory an algorithm takes,the better its space complexity is and therefore the better the algorithm is.

Hash Tables

what is a hash table?

A hash table is a key-value store.

It’s a data structure where you are able to store pairs of keys and values where every key maps to a value.

inserting a key-value pair,deleting a key-value pair or searching for a value given a key are constant time operations.

Time = O(1) avg =O(N) worst

Key can be string such as foo, bar, baz and it turns out that our keys can even be other datatypes

hash tables are built on top of arrays.

use what’s called a hash function to transform the key,(a string,)into an index. One of the easiest ways is to map every character in the string to its ASCII character encoding,

they support constant time insertion,deletion and search on average but in the worst case,(collided)will take O of n time.

in the industry and in coding interviews, you typically forgets about the O of n worst case scenario.

is if you’re in an interview question for instance where your interviewer specifically mentions you have to handle known potential edge cases and they say that you need constant time look up for instance.

May be in such a case they would really be kind of to test you on your knowledge of hash tables and the fact that they have O of n worst case scenarios.

But if that’s not the case,typically in coding interviews,you just accept that a hash table supports constant time insertions, deletions and searching on average but you almost just treat it as all the time.

Trees

Trees are one of the most prominent data structures in all of Computer Science, Software Engineering and of course, Coding Interviews.

A tree is a type of graph.

In the context of Coding Interviews,trees are actually quite simple. You just have to be familiar with how they work conceptually and then you have to practice manipulating them.

Definition

When we talk about a tree, what we’re referring to a graph structure that is rooted. Meaning it has a root node, or conceptually you can think of this as the top node of the structure.

Then the structure is also gonna be Acyclic. Meaning it’s not gonna have cycles. each node in the tree can only have one parent.

the tree is not allowed to be disconnected.

imagine a management chain in a company

Binary Tree

Binary Tree is just a normal tree where every node has at most two child nodes.

there are other types of trees. like ternary Tree. So we could give a third child node to the root node,

k-ary trees, are trees where every node has at most k child nodes.

Binary Search trees, also knows as BSTs. BSTs are special type of Binary Tree,where every node in the BST, also satisfies a special BST property.

You may have heard of Heaps, or more specifically,Min-Heaps and Max-Heaps. Min and Max-Heaps which are also by the way typically Binary Heaps, are special types of Binary trees where every node in the tree satisfies,the Min or Max Heap property.

You may have heard of trees spelt Tries. Tries are a tree-like data structure,that typically stores characters in a string and allows you to do really cool stuff with strings using a tree-like data structure.

I would encourage you to check out the questions where we have you construct each of those data structures.

for instance, constructing a BST,constructing a Min-Heap and we cover those data structures at length, in those questions

As far as space time complexity ramifications. The first one is that, as you might imagine,storing a tree, whatever type of tree it may be,is almost always gonna be an O of N space complexity where N is the total number of nodes in the tree. If you’re traversing through an entire tree.it would take O of N time.

if you’re going down the Binary Tree and at every step of the way you’re picking one of the two sub-trees to go down through. You will have a time complexity of O of log of N.

there are very advanced types of trees like red-black trees or AVL trees,that are constructed in such a way that they can rebalance themselves, to maintain this log of N complexity. The reason they exists, and the reason they’re important is because, maintaining this log of N property,is really important.

vocabulary

We say that any path in a tree that start at the root node and ends at one of the bottom nodes in the tree is a Branch.

Then these bottom nodes, we call leaf-nodes.

We say that every level in a tree so the root node in one level, this is a second level, this is a third level. All of these levels are called Levels.

We say that a tree is complete, if every single level in the tree is filled up except the final level which may or may not be filled up. But if the final level has nodes,they should be filled up from left to right,so for instance, this I’ll draw out a quick tree.

Then we say that a tree is full if every node in the tree has either no children nodes or K children nodes. Where k is the number in the k-ary tree. We would say that the tree is full if every node either has no children or exactly k-children.

Perfect Tree. A perfect tree is a tree where all of the leaf-nodes have the same depth.

Depth is basically how far down a tree goes.
