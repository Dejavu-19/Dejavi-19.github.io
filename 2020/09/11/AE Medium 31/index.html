1. Three Number Sum

  1. First make the array sorted in ascending order.
  2. Iterate the array, fix the current number, use left and right pointers to the immediately after and far end of the array, at the very end
  3. Move two pointers in the rest of the array compare to Sum - current number, find all possible combination
  4. Time O(N^2^), Space O(N), might end up storing every single number in the array

2. Smallest Difference

  1. Find a pair of number, first element from array1, second from array2 with smallest difference
  2. Solution1: One solution come to my mind which is a sort of naive method, is use two for loop to generate all different pairs number as you can, to calculate the difference and keep track the smallest one
  3. Solution2: Solve it more optimally.
    • First sort two array both in ascending order. Take advantage of the properties of the sorted array.
    • Iterating through both arrays. Put pointers at the beginning of both arrays. Compare two element and calculate the difference, if p1 < p2, increment p1, else increment p2, keep track the smallest diference
    • if we eventually arrive to the end of the both arrays and then we call quits
  4. Time O(NlogN + mlogm) (because wo use sort) Space O(1)

6. Longest Peak

  1. A peak is defined as at least three consecutive integers in the array. They are strictly increasing until they reach the some largest value amongst them, the tip of the peak, and then become strictly decreasing.
  2. Find all the peaks. From all those peaks, expand on either side and see how long the peak is. Finally compare all the length and find the biggest one.
